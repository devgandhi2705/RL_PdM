## ★ PRIMARY PAPER RESULT

**`src/train.py` → `src/evaluate.py`** — Risk-Averse CVaR QR-DQN on the full **5D** state
`[HI, HI_slope, RUL_norm, n_repairs, steps_since_replace]`, compared against
CorrectiveMaintenance / PeriodicPM / Risk-Neutral DQN.

- Checkpoint: `results/qrdqn_best.pth` (saved on lowest eval catastrophe-rate, gated
  against do-nothing collapse — see `train.py` lines 484-495)
- Evidence this is the main result: `train.py`'s own docstring calls
  `final_comparison.csv` the "4-policy comparison"; `evaluate.py`'s docstring says
  it generates "all paper figures and tables for the Risk-Averse PdM paper"; every
  downstream experiment (`rl_benchmarks.py`, `risk_analysis.py`, `repair_ablation.py`,
  `experimental_audit.py`, `state_ablation.py`, `evaluate_d3qn.py`) loads
  `qrdqn_best.pth` as the reference/baseline agent, and `final_figures.py` treats
  `training_log.csv` + `final_comparison.csv` as the canonical training/eval record.
- Confidence: **high** — this is unambiguous from the code.

## NEEDS MANUAL REVIEW

**`results/qrdqn_ep500.pth` through `results/qrdqn_ep8000.pth` (16 files)**

- No script currently in `src/` produces this filename pattern as its primary flow.
- Traced the pattern to `src/qrdqn_agent.py:619-620`, inside the legacy
  `train_qrdqn()` helper:
  `if ep % save_every == 0: agent.save_checkpoint(save_path.with_stem(f"{save_path.stem}_ep{ep}"))`
  — this is the *only* code in the repo that can produce `qrdqn_ep{N}.pth`.
- Loaded `qrdqn_ep500.pth`'s metadata directly: `state_dim=3, N_quantiles=51,
  risk_mode=cvar, opt_steps=200000, env_steps=299456`, and the checkpoint's key
  set (9 keys, no `epsilon`/`current_episode`/`cvar_alpha`) matches
  `QRDQNAgent.save_checkpoint()`'s schema from an **earlier version** of the code
  (current `save_checkpoint` writes more keys — see `qrdqn_agent.py:472-486`).
- The only current caller of `train_qrdqn()` is `experimental_audit.py` Part B,
  but it explicitly sets `save_every=n_episodes + 1` to *disable* periodic
  per-episode saves. No other script calls `train_qrdqn()` at all.
- `state_dim=3` doesn't match `train.py`/`rl_benchmarks.py`/`risk_analysis.py`/
  `repair_ablation.py` (all 5D), nor the committed `state_ablation.py` (which
  saves to `ablation_state{A,B,C}.pth`, not `qrdqn_ep*.pth`).
- Conclusion: these 16 files are leftovers from an **exploratory/manual run of a
  now-superseded `train_qrdqn()` code path** (likely predates `train.py`'s custom
  training loop or `state_ablation.py`'s existence) — possibly an early 3D-state
  prototype before `state_ablation.py` and `dueling_qrdqn.py` were written. I
  cannot find a current script that reproduces or depends on them.
- **Flagging explicitly rather than guessing which experiment folder they belong
  in.** Recommend routing to `_archive_unassigned/` unless the user recognizes
  this run and can confirm which phase it belongs to.

**`results/.gitkeep`** — not experiment-specific; a placeholder to keep the empty
dir tracked in git. Recommend `_shared_data/` or leaving at `results/` root.
